# El patrón Modelo-vista-modelo o MVVM

Se trata de un patrón que ayuda a la separación de responsabilidades entre
la interfaz de usuario y la lógica de negocios.

Vue logra este patrón generando reactividad en sus miembros.

# Vue posee algunas directivas comunes

<ul>
  <Appear>
    <li>v-if: Permite la evaluación de alguna condición ya sea un método, campo o propiedad.</li>
    <li>v-else: Debe estar en un elemento hermano inmediato a un v-if, y ya sabemos como funciona.</li>
    <li>v-for: Iteración para generar múltiples elementos en el DOM, debe estar acompañado de un key (Más adelante lo veremos).</li>
    <li>v-model: Two-way data binding, para elementos que el usuario puede modificar, asigna el valor de ese elemento a un campo de Vue y viceversa.</li>
  </Appear>
</ul>

---

<ul>
  <Appear>
    <li>v-html: Toma un contenido inseguro pre-renderizado y lo escribe. (Un post de Wordpress por ejemplo).</li>
    <li>v-bind:atributo: One-way data binding para atributos.</li>
    <li>v-on:evento: Toma un evento y asigna su ejecución a un método.</li>
    <li>v-show=”condicion”: Funciona muy similar a v-if, pero en lugar de renderizar o no el elemento, este lo muestra o lo oculta, dejándolo existente en el DOM.</li>
    </Appear>
</ul>

---

# Algunas otras directivas no tan comunes

<ul>
  <Appear>
    <li>v-text: Es similar al data binding por interpolación, funciona como v-html pero es sanitizado.</li>
    <li>v-pre: Se salta otras directivas internas y la interpolación.</li>
    <li>v-once: Ejecuta la lógica de rendereo asociada a ese elemento una única vez.</li>
    <li>v-cloak: Sencillamente es una directiva que existe sobre el elemento hasta que este termina su renderizado.</li>
  </Appear>
</ul>
